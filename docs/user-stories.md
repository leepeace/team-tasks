# User Stories

## 1. 핵심 시나리오

### P1 팀장(Task Owner) — 출근부터 퇴근까지 하루 사용 흐름

> 기술 수준: 낮음 / 툴보다 결과 중심 / 복잡한 UI 기피

---

#### 09:00 출근 — 오늘 팀 상황 파악

1. 브라우저에서 team-tasks 보드를 연다.
2. 칸반 보드 전체를 한눈에 훑어 **"진행 중"** 컬럼에 일감이 몇 개인지 확인한다.
3. 전날 "완료"로 넘어온 일감을 확인해 팀 진척을 파악한다.

---

#### 09:20 오전 스탠드업 전 — 신규 일감 등록

1. **"+ 새 일감"** 버튼을 클릭한다.
2. 제목, 담당자, 마감일, 우선순위(높음)만 입력하고 저장한다 — 30초 이내.
3. 같은 방식으로 오늘 배분할 일감 2~3개를 연속으로 추가한다.
4. 보드로 돌아와 방금 만든 일감이 **"할 일"** 컬럼에 나타나는지 확인한다.

---

#### 09:30 스탠드업 미팅 — 현황 공유

1. 담당자 필터를 **"전체"** 로 두고 보드 화면을 팀원에게 화면 공유한다.
2. 우선순위 **"높음"** 배지가 붙은 일감을 짚으며 오늘 집중 항목을 안내한다.
3. 특정 팀원의 부하가 많아 보이면 담당자 필터로 해당 인원만 걸러 재배분 여부를 결정한다.

---

#### 11:00 오전 중 — 돌발 요청 처리

1. 상위 부서에서 긴급 요청이 들어온다.
2. 새 일감을 **우선순위: 높음**으로 즉시 등록하고 담당자를 지정한다.
3. 담당자가 알아볼 수 있도록 제목에 [긴급] 접두어를 붙인다.

---

#### 13:00 점심 후 — 중간 점검

1. 보드를 열어 오전에 등록한 일감의 상태가 **"진행 중"** 으로 바뀌었는지 확인한다.
2. 아직 **"할 일"** 에 머물러 있는 일감이 있으면 담당자에게 구두로 확인한다.

---

#### 15:00 오후 — 진척 재확인 및 우선순위 조정

1. 담당자 필터로 팀원별 진행 현황을 순서대로 확인한다.
2. 마감일이 오늘인 일감이 아직 "진행 중"이면 우선순위를 **"높음"** 으로 수정한다.
3. 완료된 일감은 보드에서 **"완료"** 컬럼에 있음을 확인하고 별도 조치 없이 넘어간다.

---

#### 17:30 퇴근 전 — 일일 마감 정리

1. 상태 필터를 **"완료"** 로 걸어 오늘 처리된 일감 목록을 확인한다.
2. 완료 건수를 팀 채널에 요약 공유하기 위해 화면을 캡처한다.
3. 내일로 이월될 **"진행 중"** 일감을 훑어보고 마감일이 지난 항목이 없는지 점검한다.
4. 브라우저를 닫는다 — 별도 저장 불필요, 상태는 자동 유지.
