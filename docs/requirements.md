# Requirements

> MVP 헌장: 단일 테이블 · 기능 5개 이하 · 1일 안에 코드로 옮길 수 있는 분량

---

## 기능 요건

| ID | 이름 | 설명 | 우선순위 |
|----|------|------|----------|
| F-01 | 일감 생성 | 제목·담당자·마감일·우선순위(높음/중간/낮음)를 입력해 일감을 등록한다. 필수 입력은 제목만이며 30초 이내 완료 가능해야 한다. | 필수 |
| F-02 | 상태 변경 | 일감 상태를 할 일 → 진행 중 → 완료 순으로 클릭 한 번에 전환한다. 변경 즉시 칸반 컬럼이 갱신된다. | 필수 |
| F-03 | 칸반 보드 조회 | 전체 일감을 할 일 / 진행 중 / 완료 3개 컬럼으로 나눠 한 화면에 표시한다. 새로고침 없이 최신 상태를 유지한다. | 필수 |
| F-04 | 담당자·우선순위 필터 | 담당자 이름 또는 우선순위로 보드를 즉시 필터링한다. 필터 초기화 버튼으로 전체 보기로 돌아온다. | 필수 |
| F-05 | 일감 수정 | 등록된 일감의 제목·담당자·마감일·우선순위를 수정한다. 수정 후 보드에 즉시 반영된다. | 필수 |

---

## 비기능 요건

| 영역 | 지표 | 기준 |
|------|------|------|
| **성능** | 초기 페이지 로드 시간 | 3G 네트워크 기준 LCP 3초 이하 |
| **성능** | 상태 변경 반응 시간 | 클릭 후 UI 갱신 200ms 이하 |
| **보안** | 입력값 검증 | 모든 사용자 입력에 XSS 방지 처리 적용, 서버 액션에서 입력 길이 상한(제목 100자) 강제 |
| **가용성** | 서비스 정상 가동률 | 월 기준 99% 이상 (다운타임 7시간 12분 이하) |
| **접근성** | 키보드 탐색 | 모든 인터랙티브 요소를 Tab·Enter·Esc 키만으로 조작 가능 |
| **접근성** | 색 대비 | WCAG 2.1 AA 기준 — 텍스트 명도 대비 4.5:1 이상 |
