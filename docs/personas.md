# Team-Tasks 페르소나

> MVP 헌장: 단일 테이블 · 기능 5개 이하 · 1일 안에 코드로 옮길 수 있는 분량

| 구분 | 역할 | 기술 수준 | 주요 시나리오 |
|------|------|-----------|--------------|
| **P1** | **팀장 (Task Owner)** — 일감을 만들고 배분하는 의사결정자 | 낮음 — 툴보다 결과에 집중, 복잡한 UI 기피 | ① 신규 일감을 제목·담당자·마감일만 입력해 30초 안에 등록한다 ② 전체 칸반 보드를 한눈에 훑어 병목 컬럼을 파악한다 ③ 완료된 일감을 필터링해 주간 진척 현황을 팀원에게 공유한다 |
| **P2** | **실무자 (Assignee)** — 일감을 받아 처리하는 개발자·기획자 | 중간 — 사내 협업 툴에 익숙, 단축키·빠른 상태 변경 선호 | ① 내 이름이 할당된 일감만 필터링해 오늘 할 일 목록을 확인한다 ② 작업 시작 시 상태를 "진행 중"으로 드래그·드롭 없이 클릭 한 번으로 바꾼다 ③ 완료 후 체크리스트 항목을 체크하고 상태를 "완료"로 전환한다 |
| **P3** | **관찰자 (Stakeholder)** — 진행 상황을 모니터링하는 관련 부서·경영진 | 낮음 — 읽기 전용 접근, 별도 로그인 없이 현황 파악 희망 | ① 로그인 없이 공유 링크로 보드를 열어 전체 일감 상태를 조회한다 ② 우선순위(높음·중간·낮음) 배지로 긴급 이슈를 즉시 식별한다 ③ 특정 담당자 이름으로 필터를 걸어 해당 인원의 업무 부하를 확인한다 |
